# Refactor Assistant Skill

帮助识别需要重构的代码，并提供具体的重构方案。

## Instructions

重构分析流程：

1. **识别代码异味**
   - 长函数（>50 行）
   - 大类（>500 行）
   - 重复代码
   - 过多参数（>3 个）
   - 复杂条件判断
   - 神秘命名

2. **评估重构优先级**
   - 高：影响可读性和维护性
   - 中：影响扩展性
   - 低：代码风格问题

3. **提供重构方案**
   - 提取函数
   - 提取类
   - 引入参数对象
   - 替换条件表达式
   - 简化函数调用

4. **重构步骤**
   - 小步重构
   - 每步后运行测试
   - 保持功能不变

## Usage Examples

### Example 1: 长函数重构
**Input**: 100 行的函数处理多个职责

**Output**:
- 识别 3 个独立职责
- 提取为 3 个独立函数
- 主函数变为协调者

### Example 2: 重复代码
**Input**: 多处相似的验证逻辑

**Output**:
- 提取通用验证函数
- 使用配置驱动验证
- 减少代码重复

## Guidelines

- 重构前确保有测试覆盖
- 一次只做一个重构
- 提交前运行完整测试套件
- 保持向后兼容（公共 API）
- 记录重构原因和影响
